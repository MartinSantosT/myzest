Zest ‚Äî User Manual

Zest is a self-hosted culinary diary that stores your recipes, memories, and the stories around the table. This manual covers everything you need to get started and make the most of every feature.

1. Installation

Requirements

  • ‚Ä¢ Docker and Docker Compose installed on your machine
  • ‚Ä¢ About 200 MB of disk space (image + dependencies)
  • ‚Ä¢ Any hardware: Raspberry Pi 4, NAS, old laptop, cloud VM ‚Äî anything that runs Docker

Quick Start

git clone https://github.com/MartinSantosT/myzest.git
cd myzest
cp .env.example .env
docker compose up -d

Open http://localhost:8000 in your browser. That's it.

Secure Your Instance

  1. Change the secret key ‚Äî Open .env and replace the default value:

    # Generate a secure key:
python -c "import secrets; print(secrets.token_urlsafe(32))"

    Paste the result as ZEST_SECRET_KEY in your .env file, then restart:

    docker compose down && docker compose up -d

  2. Change the default password ‚Äî Log in with admin@zest.local / admin, then go to Settings ‚Üí Profile and update your password immediately.

Custom Port

Edit docker-compose.yml to change the port:

ports:
  - "3000:8000"   # Access Zest on port 3000

Reverse Proxy

If you're exposing Zest to your local network or the internet, place it behind a reverse proxy (Nginx, Caddy, Traefik, or Nginx Proxy Manager). See the README for an example Nginx configuration.

3. Recipes

Creating a Recipe

  1. Click + New Recipe (or the orange button in the top right)
  2. Fill in the details:
    • ‚ó¶ Title (required)
    • ‚ó¶ Description ‚Äî A short intro or story about the dish
    • ‚ó¶ Prep Time & Cook Time ‚Äî In minutes
    • ‚ó¶ Servings ‚Äî How many portions the recipe makes
    • ‚ó¶ Photo ‚Äî Click the image area to upload a hero photo. You can add up to 4 photos total (1 hero + 3 additional)
  3. Ingredients ‚Äî Add one per row. Each ingredient has structured fields:
    • ‚ó¶ Quantity (number)

Editing a Recipe

Deleting a Recipe

In the recipe detail view, click Delete (trash icon). You'll be asked to confirm. Deletion removes the recipe, its ingredients, steps, images, and any associated cookbook entries. Linked memories are preserved but their recipe association is removed.

Rating & Favorites

  • ‚Ä¢ Rating ‚Äî Click the stars (1‚Äì5) on any recipe card or in the detail view

Multi-Photo Support

Each recipe supports up to 4 photos: one hero image (displayed on the card) and 3 additional photos visible in the detail view. Click the "+" area below the hero image to add more photos.

5. Importing Recipes from URLs

Zest includes a powerful 4-tier scraping engine that can extract recipes from virtually any cooking website.

How to Import

  1. Click the "Import from URL" option in the sidebar
  2. Paste the URL of the recipe page
  3. Click Scrape
  4. Zest shows a preview with the extracted data: title, description, ingredients, steps, times, servings, and photo
  5. Review and edit anything before saving
  6. Click Save to add it to your collection

How the Scraper Works

Zest tries four methods in order, using the first one that succeeds:

  1. recipe-scrapers library ‚Äî Supports 400+ popular cooking sites with site-specific parsers
  2. JSON-LD ‚Äî Reads structured data embedded in the page (Schema.org Recipe format)
  3. Microdata ‚Äî Reads HTML microdata attributes
  4. Heuristic analysis ‚Äî Intelligent CSS selector matching for unstructured pages

Imported recipes are automatically tagged with the "Imported from Internet" category and a blue badge showing the source domain.

7. What to Cook?

This is one of Zest's unique features. It answers the question: "I have chicken, rice, and onion ‚Äî what can I make?"

How to Use

  1. Click "What to Cook?" in the sidebar
  2. Start typing an ingredient name ‚Äî suggestions appear from your recipe database
  3. Click to add ingredients to your list
  4. Zest instantly shows matching recipes, ranked by match percentage

Understanding the Results

Each result shows:

  • ‚Ä¢ Match percentage ‚Äî How many of the recipe's ingredients you have (green = 75%+, orange = 50%+, gray = below 50%)
  • ‚Ä¢ Missing ingredients ‚Äî Highlighted in red so you know what you'd need to buy
  • ‚Ä¢ Portion adjustment ‚Äî Scale the recipe right from the results
  • ‚Ä¢ "Add to Shopping List" ‚Äî One click to add the missing ingredients to your shopping list

9. Memories

Memories are what makes Zest different from every other recipe manager. A Memory connects photos, dates, locations, and stories to a recipe ‚Äî turning your recipe collection into a culinary diary.

Creating a Memory

  1. Click Memories in the sidebar
  2. Click + New Memory
  3. Fill in the details:
    • ‚ó¶ Title ‚Äî "Christmas dinner 2025", "First attempt at sourdough", etc.
    • ‚ó¶ Photos ‚Äî Upload up to 10 photos. JPEG photos with EXIF data will auto-fill the date and location.
    • ‚ó¶ Date ‚Äî When it happened (auto-detected from photo EXIF if available)
    • ‚ó¶ Location ‚Äî Where it happened (auto-detected from GPS coordinates via reverse geocoding)
    • ‚ó¶ Story ‚Äî The description of what happened, who was there, why it was special
    • ‚ó¶ Linked Recipe ‚Äî Associate this memory with a recipe from your collection
  4. Click Save

Photo Features

  • ‚Ä¢ Multi-photo upload ‚Äî Select multiple photos at once
  • ‚Ä¢ EXIF auto-detection ‚Äî Zest reads the date and GPS coordinates from JPEG photos automatically
  • ‚Ä¢ Reverse geocoding ‚Äî GPS coordinates are converted to readable place names (e.g., "Brooklyn, New York")
  • ‚Ä¢ HEIC/HEIF support ‚Äî iPhone photos work out of the box
  • ‚Ä¢ Image optimization ‚Äî Photos are automatically resized (max 1920px) and converted to JPEG for optimal storage

Viewing Memories

  • ‚Ä¢ Grid view ‚Äî All memories displayed as cards with cover photo, date, location, and linked recipe
  • ‚Ä¢ Detail view ‚Äî Click any memory to see the full photo gallery, story, and metadata
  • ‚Ä¢ From recipe view ‚Äî Each recipe shows its associated memories in a dedicated section

Editing & Deleting

  • ‚Ä¢ Click the Edit button in the memory detail view to modify any field or add/remove photos
  • ‚Ä¢ Click Delete to permanently remove the memory and its photos

15. Backups & Data

Automatic Backups

  1. Go to Settings ‚Üí Backups
  2. Enable automatic backups
  3. Configure:
    • ‚ó¶ Frequency ‚Äî Every 12 hours, 24 hours, or 7 days
    • ‚ó¶ Retention ‚Äî How many backups to keep (default: 7)
    • ‚ó¶ Include images ‚Äî Whether to include uploaded photos in backups

Backups are stored in the ./data/backups/ directory on your host machine.

Manual Export

  • ‚Ä¢ Full backup (ZIP) ‚Äî Exports the complete database + all images. Use this for disaster recovery or migrating to a new server.
  • ‚Ä¢ JSON export ‚Äî Exports recipes in a portable JSON format for interoperability.

Import

  • ‚Ä¢ Database import ‚Äî Upload a full backup ZIP to restore your entire instance. Warning: this replaces the current database completely.
  • ‚Ä¢ JSON import ‚Äî Upload a JSON file to add recipes to your existing collection. Duplicates are detected and skipped.

Migration Between Servers

  1. Export a full backup from the old server
  2. Set up a fresh Zest instance on the new server
  3. Import the backup
  4. Done ‚Äî all recipes, memories, cookbooks, and images are restored

18. Troubleshooting

"Port 8000 is already in use"

Change the port in docker-compose.yml:

ports:
  - "9000:8000"

On Windows, ports 7681‚Äì8782 may be blocked by Hyper-V. Use port 9000 or higher.

Photos won't upload

  • ‚Ä¢ Check that the ./app/static/uploads directory exists and is writable
  • ‚Ä¢ Maximum file size depends on your reverse proxy configuration. Add client_max_body_size 20M; to your Nginx config.

Database migration errors

Run the migration script manually:

docker exec zest_backend python migrate.py

The script is idempotent (safe to run multiple times) and creates a backup before making changes.

HEIC photos not working

HEIC support requires the pillow-heif package, which is included in the default Docker image. If you're running Zest outside Docker, install it separately:

pip install pillow-heif

Forgot your password

If you're locked out, you can reset via the database:

docker exec -it zest_backend python -c "
import bcrypt
from app.database import SessionLocal
from app.models import User
db = SessionLocal()
user = db.query(User).filter(User.email == 'your@email.com').first()
user.password_hash = bcrypt.hashpw(b'newpassword', bcrypt.gensalt()).decode()
db.commit()
print('Password reset successfully')
"
